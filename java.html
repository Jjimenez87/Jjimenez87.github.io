Programming Paradigms

Programming Paradigms are ways or styles of writing code, their are not programming languages.

Classify programming languages based on the paradigms they support:

  • Smalltak, support a single paradigm (OOP)
  • Ruby,Python,Java, JavaScript support multiple paradigms

In computer science we have various programming paradigms, or styles of programming.

like procedural, functional, object-oriented, event-driven, logic, aspect-oriented, and so on.

Out of all these object-oriented and functional are the 2 most popular paradigms we use these days

Object-Oriented Programming

In OOP, everything is based on the concept of objects

these objects interact with each other to perform various tasks

These objects are units that contain some data (state), and operations on the data (methods/behaviour)

These are the methods or functions that modify the data

what programming paradigm is the best? The answer is, it depends on the problem, you're trying to solve

Both object-oriented and functional programming are great and have benefits. But neither work best in all situations

One of the major issue we have in the software industry these days is that developers are more excited about languages, tools, frameworks

than solving problems, problem solving is the process of defining a problem, identifying and comparing different solutions,

and picking the one that best solves that problem with respect to the context and constraints.

Take some poorly written code and Refactor it into clean and maintainable, reusable and object oriented code is a critical skill that

every Java Developer must have

Benefits of OOP

  • Reduced Complexity
  • Easier Maintenance
  • Code Reuse
  • Faster Development
